## Overview

Each ClawControl deployment is configured using a `config.json` file stored in `~/.clawcontrol/deployments/<deployment-name>/`. This file defines your cloud provider settings, OpenClaw configuration, and agent settings.

## Configuration Schema

The deployment configuration follows the `DeploymentConfigSchema` defined in `src/types/index.ts:167`.

### Core Fields

<ParamField path="name" type="string" required>
  Unique identifier for the deployment. Must be lowercase letters, numbers, and hyphens only. Must start and end with a letter or number. Maximum 63 characters (RFC 1123 hostname format).
</ParamField>

<ParamField path="provider" type="enum" required>
  Cloud provider to use for deployment.
  
  **Options:** `hetzner`, `digitalocean`, `vultr`
</ParamField>

<ParamField path="createdAt" type="string" required>
  ISO 8601 timestamp of when the deployment was created.
</ParamField>

<ParamField path="skipTailscale" type="boolean">
  Whether to skip Tailscale installation and configuration. Set to `true` to deploy without Tailscale VPN.
  
  **Default:** `false`
</ParamField>

### Provider-Specific Configuration

<ParamField path="hetzner" type="object">
  Hetzner-specific configuration. Required when `provider` is `hetzner`.
  
  <Expandable title="properties">
    <ParamField path="hetzner.apiKey" type="string" required>
      Hetzner Cloud API key for authentication.
    </ParamField>
    
    <ParamField path="hetzner.serverType" type="string">
      Server type/size to provision.
      
      **Default:** `cpx11` (2 vCPU, 2 GB RAM)
    </ParamField>
    
    <ParamField path="hetzner.location" type="string">
      Datacenter location code.
      
      **Default:** `ash` (Ashburn, VA)
    </ParamField>
    
    <ParamField path="hetzner.image" type="string">
      Operating system image.
      
      **Default:** `ubuntu-24.04`
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="digitalocean" type="object">
  DigitalOcean-specific configuration. Required when `provider` is `digitalocean`.
  
  <Expandable title="properties">
    <ParamField path="digitalocean.apiKey" type="string" required>
      DigitalOcean API token for authentication.
    </ParamField>
    
    <ParamField path="digitalocean.size" type="string">
      Droplet size slug.
      
      **Default:** `s-1vcpu-2gb`
    </ParamField>
    
    <ParamField path="digitalocean.region" type="string">
      Datacenter region slug.
      
      **Default:** `nyc1`
    </ParamField>
    
    <ParamField path="digitalocean.image" type="string">
      Operating system image slug.
      
      **Default:** `ubuntu-24-04-x64`
    </ParamField>
  </Expandable>
</ParamField>

### OpenClaw Configuration

<ParamField path="openclawConfig" type="object">
  Optional configuration overrides for OpenClaw gateway and browser settings. See [OpenClaw Configuration](/configuration/openclaw-config) for details.
</ParamField>

<ParamField path="openclawAgent" type="object">
  AI agent and channel configuration. See [OpenClaw Configuration](/configuration/openclaw-config#agent-configuration) for details.
</ParamField>

## Storage Location

Configuration files are stored at:

```
~/.clawcontrol/deployments/<deployment-name>/config.json
```

The directory structure is managed by `src/services/config.ts:12-13`:

```typescript
const CLAWCONTROL_DIR = join(homedir(), ".clawcontrol");
const DEPLOYMENTS_DIR = join(CLAWCONTROL_DIR, "deployments");
```

```json
      "defaultProfile": "openclaw"

  "openclawAgent": {
    "aiProvider": "openrouter",
    "aiApiKey": "sk-or-v1-...",

  "skipTailscale": false
}
```
